# 🦄 SweetyAI 投資企劃書：情感連結與生活入口的 AI 革命

> **核心願景**：不僅是 AI 助理，更是介入真實人際關係的「傳話天使」與「生活萬能管家」。透過最普及的通訊軟體 (LINE/WhatsApp)，為 50+ 熟齡世代打造零門檻的科技生活入口。

---

## 1. 專案概述 (Executive Summary)

*   **產品名稱**：SweetyAI
*   **產品定位**：
    1.  **人際潤滑劑**：替用戶優化語氣、傳遞情感，真實介入並改善人際關係。
    2.  **生活入口 (Life Portal)**：從聊天到辦事，一句話解決生活大小事。
*   **目標市場**：**45-65+ 熟齡世代**。
    *   **痛點**：不擅長打字、不懂如何委婉表達情感、不想學習新 App 的操作介面。
    *   **優勢**：直接建立在 LINE/WhatsApp 內，**完全無需學習新介面**，像傳訊息給朋友一樣簡單。
*   **短期目標**：12 個月內獲取 10 萬活躍用戶。

---

## 2. 產品優勢與差異化 (Competitive Advantage)

### ✅ 對 50+ 極度友善 (Zero Learning Curve)
*   不同於 ChatGPT 或其他 AI App 需要下載並學習新操作。
*   SweetyAI 直接「住」在用戶最熟悉的 LINE 裡。用戶只需會傳訊息，就能使用最強大的 AI 科技。

### ✅ 獨特功能：人際關係的中介者 (Relationship Intermediary)
*   SweetyAI 不只是陪聊，而是能「替用戶傳話」。
*   **場景舉例**：
    *   用戶：「叫我女兒滾回來吃飯！」
    *   AI (轉換後傳給女兒)：「女兒啊，這週末有空嗎？媽媽想做頓好吃的等你回來聚聚，如果忙也沒關係，以工作為重喔！」
    *   **價值**：消除代溝，讓溝通更圓滑溫暖。

### ✅ 通訊軟體的低成本特性
*   在通訊軟體上的對話通常短小精簡 (Short context)，鮮少出現長篇大論。
*   相較於網頁版生成式 AI，**Token 用量極低**，大幅降低營運成本。

## 3. 獲利模式 (Revenue Model) - 雙階段策略

**核心策略：初期完全免訂閱費 (Free-to-Use)，透過廣告變現；中後期轉型為高單價服務入口。**

### 🚀 第一階段：流量與廣告變現 (初期)
用戶端完全免費，但在 Token 用量上做出合理限制 (如每日免費額度)。

1.  **主動式廣告 (Active Ads)**：
    *   利用 LINE 推播功能，精準投放合作夥伴的優惠訊息 (如：長輩營養品、旅遊行程)。
2.  **被動式廣告 (Passive Ads)**：
    *   當用戶主動詢問相關資訊時置入。
    *   *例：用戶問「最近有什麼好劇？」 -> AI 推薦 Netflix 韓劇並附上相關週邊導購。*
3.  **回應式廣告 (Responsive Ads)**：
    *   類似 ChatGPT Search，當對話進入特定情境時自然觸發。
    *   *例：用戶聊到「膝蓋痛」 -> AI 溫馨提醒保養知識，並自然推薦「維骨力」或「復健診所」。*

### ❤️ 第二階段擴充：Agent 社交與情感連結 (Matchmaking)
*   **Agent 社交廳 (Agent Social Lounge)**：
    *   **概念**：不只是用戶找對象，而是讓用戶的 **AI Agent (分身)** 在虛擬空間進行社交。
    *   **流程**：
        1.  系統自動媒合符合條件的 Agent (如：喜歡登山的退休教師 Eric vs. 愛攝影的公務員 Judy)。
        2.  雙方 Agent 進行「模擬對話」，確認彼此主人的契合度。
        3.  AI 向主人回報：「Boss，我剛認識一位 Judy 的小秘書，我們聊了一下發現你們都喜歡音樂，這是她的 ID...」
    *   **價值**：
        *   **降低社交恐懼**：由 AI 先行試探，避免直接被拒絕的尷尬。
        *   **精準配對**：AI 深度分析對話風格與價值觀，比傳統硬性條件篩選更準確。
        *   **增加黏著度**：用戶會期待 Agent 每天「出去交際」帶回來的趣事與新朋友。

### 🚀 第三階段：生活入口與代客服務 (中後期)
當黏著度建立後，SweetyAI 將成為用戶的**「生活採購經理」**。用戶不再需要打開蝦皮、PChome 或訂票 App。

*   **代客訂購與服務 (Concierge Service)**：
    *   **場景**：用戶：「老闆要我買一台文書電腦，預算兩萬，我不懂電腦，你幫我買。」
    *   **AI 行動**：分析需求 -> 挑選最佳 CP 值機種 -> 確認收件資訊 -> **直接完成下單**。
*   **利潤來源**：
    *   商品導購分潤 (Affiliate Marketing)。
    *   代購服務費。
    *   B2B 優先推薦費。

## 4. 核心成本結構分析 (Cost Structure)

以 **10 萬用戶** 為目標，因通訊軟體特性，Token 成本較低，重點在於獲客與平台費用。

### 🤖 A. AI API 成本 (OpenAI / LLM)
*   **特性**：通訊軟體訊息短，平均單次對話僅需 300-500 tokens。
*   **預估**：
    *   活躍用戶 (1萬人)：每日 20 則 = $0.06/日 -> $1,800/月。
    *   輕度用戶 (9萬人)：每日 2 則 = $0.006/日 -> $1,620/月。
    *   **月總計**：約 **USD 3,500 (約 TWD 112,000)** - *遠低於網頁版應用*。

### ☁️ B. 雲端與平台費用 (Cloud & LINE)
*   **GCP 基礎建設**：App Server, Cloud SQL, Storage。預估 **TWD 50,000 /月**。
*   **LINE 官方帳號費用**：
    *   大部分對話使用免費 Reply API。
    *   主動推播廣告 (Push API) 需付費，但此部分成本將由**廣告收入覆蓋**。
    *   預估基礎月費與額外訊息費：**TWD 40,000 /月**。

### 👥 C. 人力與推廣
*   **人力團隊** (後端、全端、行銷)：預估 **TWD 350,000 /月**。
*   **行銷廣告投放**：針對 50+ 族群投放 (Facebook, LINE Ad)。預估 **TWD 400,000 /月**。
